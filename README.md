# Deepfake Detection via Multi-Layer Local Intrinsic Dimensionality

This experimental project detects deepfake images and videos by measuring how the **local intrinsic dimension (LID)** of feature representations changes across the depth of a Vision Transformer — without retraining the backbone.

## Why LID?

Most deepfake detectors fine-tune a classifier directly on top of visual features. This project takes a different angle: rather than asking *what* a face looks like, it asks *how the geometry of its representations is structured* inside the model.

My hypothesis is that deepfake images generated by GANs or diffusion models occupy a different manifold than real photographs. That difference is measurable as a shift in local intrinsic dimension — the effective dimensionality of the neighborhood around each sample's embedding. Real images tend to cluster in higher-dimensional, more complex manifolds; generated images collapse to lower-dimensional, more regular ones.

## Approach

LID is estimated with a *k*-NN log-ratio (hill) estimator (Amsaleg et al., 2015), and TwoNN (Facco et al., 2017) is used for diagnostic global-dimension logging. The per-point *k*-NN log-ratio feature vector is what a shallow MLP learns to discriminate.

**Reference bank.** LID is a property of a point relative to a data manifold, so it is measured against a **fixed reference bank of real-image features** — built once from real images and frozen. Each query's *k* nearest neighbours are drawn from that bank, which makes an image's LID feature a deterministic function of the image alone, single images included.

The reference bank is part of the fixed *feature transform*, not the trained classifier — analogous to k-NN exemplars or normalization statistics fit on training data. The MLP only ever sees LID distances, never reference labels. The reference images are a held-out subset of real images, disjoint from the images being classified.

LID is computed at **three depths** of the ViT encoder:

```
ViT (frozen)  →  start / middle / end layer CLS tokens
                        ↓ LID vs. fixed real-image reference bank (k-dim each)
              concat (3k-dim) → MLP → Real / Fake
```

The backbone (`prithivMLmods/Deep-Fake-Detector-v2-Model`) is fully frozen; only the MLP head is trained, with early stopping on validation AUC. Training data is [`prithivMLmods/Deepfake-vs-Real-60K`](https://huggingface.co/datasets/prithivMLmods/Deepfake-vs-Real-60K) — roughly 30k real and 30k deepfake images, mapped to `0=Real, 1=Fake`.

## Differentiation

| Aspect | Typical deepfake detectors | This project |
|--------|---------------------------|--------------|
| What is learned | Visual artifacts, texture | Geometric structure of representations |
| Backbone usage | Fine-tuned end-to-end | Frozen; only geometry is extracted |
| Feature type | Class logits or pixel features | Intrinsic dimension across layers |
| Generalization hypothesis | Artifact-specific | Manifold-level, architecture-agnostic |

The bet is that LID captures a more fundamental signal than surface-level artifacts. Whether that signal is genuinely **more robust across unseen deepfake generators** is a *hypothesis, not yet a result* — see Limitations.

## Workflow

```
train.py        build real-image reference bank + train the MLP head
                (multi-epoch, early stopping on val AUC, ViT frozen)
                loads the gated HF dataset (huggingface-cli login first)
                → saves checkpoints/lid_classifier.pth (classifier + reference bank)

                # quick smoke test: python train.py --epochs 1

inference.py    load checkpoint + run on ./data/ (flat images/videos)
                → face detection (dlib), video frame sampling
                → per-face LID inference
                → submission.csv
```

## Limitations & future work

- **Cross-generator generalization is unverified.** The central claim —
  manifold-level, generator-agnostic detection — requires a
  leave-one-generator-out evaluation (train on a subset of generators, test on
  held-out ones, against a CLS-feature baseline). That experiment is **not yet
  run** due to compute constraints; current results are in-distribution only.
  Treat the generalization framing as a motivating hypothesis to be tested.
- The reference bank is drawn from a single real-image distribution; its
  effect on calibration across domains is not yet characterized.
